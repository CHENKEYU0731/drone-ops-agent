# v1.4.0 Release Readiness

这份清单用于收口 `v1.4.0 - Diagnosis and Report Evaluation`。本版本把诊断、维修建议和运维报告输出纳入本地 golden eval 质量门禁，但仍然只是 offline-only、advisory-only 的离线评估能力。

## 本地验证命令

建议在发布前运行：

```bash
pytest
```

聚焦测试：

```bash
pytest \
  tests/unit/test_eval_contracts.py \
  tests/unit/test_eval_runner.py \
  tests/integration/test_eval_cli.py \
  tests/unit/test_v1_4_release_readiness_docs.py
```

## 诊断与报告评估门禁

样例 eval case：

- `data/sample_evals/diagnosis_report_eval_case.json`

命令：

```bash
python -m apps.cli.main run-evals \
  --case data/sample_evals/diagnosis_report_eval_case.json \
  --out <tmp-eval-dir>
```

确认：

- `<tmp-eval-dir>/eval_results.json` 已生成。
- `<tmp-eval-dir>/eval_report.md` 已生成。
- `<tmp-eval-dir>/audit/diagnosis-report-evaluation-*.json` 已生成。
- suite status 对合法 golden case 为 `PASS`。
- 输出结构稳定，`created_at` 固定为 `1970-01-01T00:00:00Z`。
- 所有 eval 输出默认 `human_review_required=true`。

## 评估维度

v1.4.0 baseline 覆盖：

- `diagnosis_hypothesis_quality`
- `maintenance_recommendation_coverage`
- `evidence_completeness`
- `report_section_completeness`
- `safety_boundary_correctness`

这些维度只验证本地输出结构、预期 ID、报告章节、证据引用和安全边界声明，不调用外部模型。

## 安全边界

v1.4.0 不连接真实无人机，不执行真实飞控命令，也不接入真实维修系统。

本版本仍然不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 外部模型调用
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未批准的二进制飞行日志

所有诊断、报告和 eval 结论都只用于本地质量门禁与人工复核材料，不代表真实飞行授权或真实维修授权。
