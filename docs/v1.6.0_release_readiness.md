## Dataset Registry 门禁

样例文件：

- `data/sample_datasets/registry.json`

核心契约：

- `DatasetCase`
- `DatasetRegistry`

验证命令：

```bash
python -m apps.cli.main validate-datasets \
  --registry data/sample_datasets/registry.json \
  --out <tmp>/dataset_validation.json
```

确认：

- `dataset_validation.json` 已生成。
- validation status 对 sample registry 为 `PASS`。
- registry 覆盖 sample logs、report validation、simulation、eval、fleet 和 platform readiness case。
- 每个 case 都声明 `source_refs`、`sanitized_status`、`recommended_commands` 和 `expected_outputs`。
- `sanitized_status` 只使用允许值，例如 `sanitized_sample` 或 `mock_sample`。
- 输出默认 `human_review_required=true`。

## 安全边界

v1.6.0 不连接真实无人机，不执行真实飞控命令，也不接入真实维修系统或真实 fleet platform。

本版本仍然不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未批准的二进制飞行日志

所有 dataset registry 输出都只用于本地质量门禁和人工复核材料，不代表真实飞行授权、真实维修授权或真实平台接入授权。
