# v1.7.0 - Offline Adapter and Approval Workflow Baseline

Date: draft

Release status: draft

Release positioning: offline adapter registry and approval workflow validation on top of the v1.6.0 dataset registry release.

## 范围

v1.7.0 把 v1.5.x platform readiness 中的 offline adapter contract 和 reviewer / approval model 进一步收口为可运行的本地质量门禁。

本版本仍然是 offline-only、advisory-only，不连接真实无人机、不执行飞控命令、不接入真实维修系统或真实 fleet platform。

## Offline Adapter Registry Validation

新增能力：

- 新增 `packages.adapter_workflows`。
- 新增 `validate-adapters` CLI。
- 输出 deterministic `adapter_validation.json`。
- 检查 adapter registry 的 offline/advisory safety boundary。
- 检查 adapter 禁止 `api_call`、`auto_dispatch` 和 `mavlink_command`。
- 检查 adapter 不连接真实平台，并默认需要人工复核。

## Approval Workflow Validation

新增能力：

- 新增 `ApprovalPacket` schema。
- 新增 sample fixture：`data/sample_approvals/approval_packet.json`。
- 新增 `validate-approvals` CLI。
- 输出 deterministic `approval_validation.json`。
- 检查 required reviewer roles 是否被 approval 覆盖。
- 检查 approval reviewer、role、decision 和 rationale。
- 所有 approval validation 输出默认 `human_review_required=true`。

## 安全边界

v1.7.0 不包含：

- 真实无人机连接。
- MAVLink command execution。
- arm/disarm。
- takeoff / landing / RTL。
- mission execution。
- motor start。
- firmware upload。
- flight-controller parameter writing。
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接。
- 真实 fleet platform API 调用。
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用。
- 自动派单。
- 自动执行维修动作。
- 真实、敏感或未批准的二进制飞行日志。
