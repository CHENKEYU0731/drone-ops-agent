## 概述

v0.5.0 在 v0.4.0 PDF 报告导出版之后，新增统一日志格式扩展和离线仿真验证能力。系统现在可以在保持 offline advisory 安全边界的前提下，对 CSV、JSON、PX4 ULog mock/fixture、ArduPilot BIN mock/fixture 进行统一日志分析，并支持导入 offline/mock simulation result 生成仿真验证结论、证据链和审计记录。

## 主要新增能力

- 统一 parser adapter / registry。
- `analyze-log --format auto`。
- CSV / JSON / PX4 ULog / ArduPilot BIN 格式识别。
- PX4 ULog 离线 parser 支持。
- ArduPilot BIN 离线 parser 支持。
- optional extras：`.[px4]`、`.[ardupilot]`、`.[sim]`。
- 缺少可选依赖时的清晰安装提示。
- `validate-simulation` CLI。
- offline/mock simulation result validation。
- `simulation_run.json`。
- simulation-validation audit。
- `evidence_refs` 和 `human_review_required` 约束。

## 安全边界

本版本仍为 offline advisory agent，不包含任何真实无人机控制能力：

- 不执行 MAVLink command。
- 不 arm/disarm。
- 不 takeoff/land/RTL。
- 不 mission upload / mission execution。
- 不 motor start。
- 不 firmware upload。
- 不 flight-controller parameter write。
- 不启动或连接真实 PX4 / ArduPilot / Gazebo。

PX4 ULog、ArduPilot BIN 和 offline SITL validation 仅用于本地日志分析、mock/offline 仿真结果导入、风险评估、报告生成和人工复核辅助，不代表真实飞行授权。

## 当前限制

- ULog / BIN 初版以最小字段映射和 mock/fixture 验证为主。
- SITL 初版是 offline simulation result import and validation，不是真实仿真启动器。
- 仍未接入真实硬件、工单系统或 Web dashboard。
