# v0.8.0 - Report Integration Plus

Date: 2026-06-26

Release status: ready

Release positioning: offline report integration improvements on top of the
v0.7.0 quality gates.

## 范围

v0.8.0 把日志解析、异常、诊断、维护建议、离线仿真验证和审计信息更完整地纳入单次运维复盘报告。

本版本仍然是 offline-only、mock-import-only、advisory-only 的报告增强版本，不是飞控集成版本。

Release commit:

- `56558703ea23abd1856580ea90958991e993a0b0`

## Report Simulation Validation Section Baseline

`generate-report` 现在可以选择性纳入离线仿真验证结果。

新增能力：

- `generate-report --simulation <simulation_run.json>`
- Markdown 运维报告新增 `## 7.5 仿真验证` 章节。
- 报告展示 scenario ID、仿真状态、人工复核要求、规则命中详情和 evidence refs。
- 证据引用附录会纳入 `simulation_run.json` 的仿真证据。
- `ops-report-generation` audit input refs 会记录传入的 `simulation_run.json`。

兼容性：

- 未提供 `--simulation` 时，现有报告生成行为保持不变。
- 新章节插入在安全说明之前，不改变现有 `## 10. 审计记录` 和 `## 11. 证据引用附录` 标记。

## Report Audit Metadata and Review Checklist Baseline

Markdown 运维报告现在更适合作为人工复盘材料。

新增能力：

- 报告新增 `## 7.6 审计摘要`。
- 报告新增 `## 7.7 日志解析元数据`。
- 报告新增 `## 7.8 人工复核清单`。
- `generate-report` 会读取同一输出目录下已有的 `audit/*.json`，并按稳定顺序纳入报告。
- 日志解析元数据展示 requested format、actual format、parser name、parser version、warnings 和 parser metadata。
- 人工复核清单覆盖异常、故障假设、维护建议和可选离线仿真结论。

兼容性：

- 不新增额外 CLI 参数。
- 不改变现有结构化 JSON 输出。
- 不改变 `validate-report` 需要识别的 `## 10. 审计记录` 和 `## 11. 证据引用附录` 标记。

## Report PDF Section Coverage Baseline

`generate-report --pdf` 已验证会同步包含 v0.8.0 新增报告章节。

覆盖内容：

- `## 7.5 仿真验证`
- `## 7.6 审计摘要`
- `## 7.7 日志解析元数据`
- `## 7.8 人工复核清单`
- 仿真规则 evidence refs，例如 `SIM_RESULT_COMPLETED`
- parser metadata，例如 `csv-json-flight-log@1.1.0`

本版本不重写 PDF exporter，不引入高级排版，只补充 Markdown 内容和 PDF 有效生成验证，确保新增章节不会在 `generate-report --pdf` 输出路径中回归。

## 安全边界

v0.8.0 继续保持离线建议系统定位。

本版本不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- 真实 PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接

所有报告、仿真验证和维护建议仍必须由人工复核，不代表真实飞行授权。
