# v0.6.0-draft - Real Sample Log Validation

状态：草稿，尚未发布
日期：2026-06-25

基线版本：v0.5.0

## 概述

v0.6.0 是 Real Sample Log Validation 基础设施版本。本版本不强行加入来源不明或未脱敏的真实二进制日志，而是先建立 parser coverage 文档、fixture policy、placeholder 目录、fake real-path tests 和更明确的 parser metadata，为后续安全接入小型真实/脱敏 `.ulg` / `.bin` 样例做准备。

## 新增能力

- 新增 `docs/log_parser_coverage.md`，说明 CSV、JSON、PX4 ULog 和 ArduPilot BIN 的字段映射与限制。
- 新增 `data/sample_logs/README.md`，定义真实/脱敏日志 fixture 提交策略。
- 新增 `data/sample_logs/px4/` placeholder 目录和 README。
- 新增 `data/sample_logs/ardupilot/` placeholder 目录和 README。
- 新增 fake real-path tests，覆盖 PX4 `pyulog` 路径和 ArduPilot `pymavlink` DataFlash 路径。
- 新增 optional real fixture skip 策略，不要求仓库内置真实 `.ulg` / `.bin`。
- 增强 parser metadata：
  - PX4 real path 记录 `sample_count`。
  - ArduPilot real path 记录 `messages_consumed` 和 `snapshot_count`。
- 保留现有 mock `.ulg` / `.bin` fixture。

## 未加入的内容

- 未加入未经确认的真实 PX4 `.ulg`。
- 未加入未经确认的真实 ArduPilot `.bin`。
- 未改变 parser registry 架构。
- 未把 `pyulog` 或 `pymavlink` 加入默认 / dev 依赖。

## 验证计划

- `pytest`
- `run-mvp`
- `analyze-log --format auto` CSV
- `analyze-log --format auto` JSON
- `analyze-log --format auto` PX4 mock `.ulg`
- `analyze-log --format auto` ArduPilot mock `.bin`
- `validate-simulation`
- `export-pdf`
- optional dependency 缺失错误提示验证
- 安全边界关键词扫描

## 安全边界

本版本仍然是 offline advisory agent，不包含任何真实无人机控制能力：

- 不连接真实无人机。
- 不执行 MAVLink command。
- 不 arm/disarm。
- 不 takeoff/land/RTL。
- 不 mission upload / mission execution。
- 不 motor start。
- 不 firmware upload。
- 不 flight-controller parameter write。
- 不启动或连接真实 PX4 / ArduPilot / Gazebo。

## 当前限制

- 没有内置真实二进制日志；后续需要用户提供来源明确、可公开、已脱敏、小体积样例。
- PX4 ULog 和 ArduPilot BIN 仍是最小字段映射，不代表完整飞控日志覆盖。
- ArduPilot real DataFlash path 的 `link_quality_pct` 暂时使用默认值并记录 warning。
- 真实 fixture 测试在样例不存在时 skip。

## 后续接入真实/脱敏样例步骤

1. 确认日志来源和授权。
2. 截取小于 256 KB 的最小片段。
3. 移除或扰动 GPS、序列号、操作者、客户、地点、任务名称等敏感信息。
4. 将 PX4 `.ulg` 放入 `data/sample_logs/px4/`。
5. 将 ArduPilot `.bin` 放入 `data/sample_logs/ardupilot/`。
6. 更新对应 README 中的来源、脱敏、字段覆盖和限制说明。
7. 运行 parser coverage、CLI 和 audit/evidence 验证。
