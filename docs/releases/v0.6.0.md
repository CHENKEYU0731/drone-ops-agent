日期：2026-06-25

## 概述

v0.6.0 在 v0.5.0 的 PX4 ULog、ArduPilot BIN 和 offline SITL validation 基础上，补齐真实样例日志进入仓库前所需的文档、策略、placeholder 路径、测试语义和覆盖说明。

本版本不新增真实 `.ulg` / `.bin` 二进制样例，也不声称已经完成真实日志验证。当前重点是让后续可公开、已脱敏、小体积真实样例能够按清晰标准安全接入。

## 与上一版本关系

- `v0.5.0` = Log Format Expansion and Offline SITL Validation
- `v0.6.0` = Real Sample Log Validation infrastructure

## Fixture 接收策略

后续真实/脱敏 `.ulg` / `.bin` 样例进入仓库前必须满足：

- 来源明确，并允许公开提交或已获得授权。
- 单文件建议小于 256 KB。
- 已截取为 parser 验证所需的最短片段。
- 已移除或脱敏 GPS 坐标、真实地点、操作者姓名、客户名称、机体序列号、无线链路标识、任务名称等敏感信息。
- 文件名不暴露个人、客户、地点、事件或机体身份。
- 在格式专用 README 中记录来源类型、脱敏方式、字段覆盖范围、已知限制和授权状态。

如果无法确认授权或脱敏状态，不得提交该日志。

## 安全边界

本版本仍然是 offline advisory agent，不包含任何真实无人机控制能力：

- 不连接真实无人机。
- 不执行 MAVLink command。
- 不 arm/disarm。
- 不 takeoff/land/RTL。
- 不 mission upload / mission execution。
- 不 motor start。
- 不 firmware upload。
- 不 flight-controller parameter write。
- 不启动或连接真实 PX4 / ArduPilot / Gazebo。

PX4 ULog 和 ArduPilot BIN 仍仅作为本地离线日志文件读取。

## 当前限制

- 当前仓库没有内置真实二进制日志样例。
- PX4 ULog 和 ArduPilot BIN 仍是最小字段映射，不代表完整飞控日志覆盖。
- ArduPilot real DataFlash path 的 `link_quality_pct` 暂时使用默认值并记录 warning。
- 真实 fixture 测试在样例不存在时 skip。
