# v0.7.0 - Report Validation and Evidence Index Baseline

日期：2026-06-25

Release 名称：`v0.7.0 - Report Validation and Evidence Index Baseline`

版本定位：离线报告证据链质量门禁基线

## 概述

v0.7.0 的第一个小 PR 强化 `validate-report` 工作流，让 `run-mvp` 生成的结构化 JSON、Markdown 报告和 audit JSON 可以被自动检查。该能力用于离线可信度检查、审计辅助和回归测试，不代表真实飞行安全许可。

## 主要新增能力

- 新增 `packages/report_validation/` 报告验证模块。
- 新增 `drone-ops validate-report` CLI。
- 支持 `--report-dir <dir>` 一键验证目录。
- 支持显式传入 `flight_summary.json`、`anomalies.json`、`diagnosis.json`、`maintenance_recommendations.json`、`ops_report.md` 和 `audit` 目录。
- 支持 `--write-index` 写出：
  - `evidence_index.json`
  - `report_validation.json`
- `evidence_index.json` 使用稳定 key 汇总 evidence refs。
- `report_validation.json` 使用确定性结构，便于 golden/snapshot 风格测试。
- validation findings 会暴露缺失 evidence refs、断裂 evidence reference、缺失关键 audit、缺失报告章节等问题。

## 验证规则

- anomaly 必须包含非空 `evidence_refs`。
- anomaly evidence ref 必须包含 source、field、rule 和 description 等关键字段。
- diagnosis 必须有 evidence，并能追溯到 summary 或 anomaly evidence。
- high/critical diagnosis 必须 `human_review_required=true`。
- maintenance recommendation 必须有 evidence，并能追溯到 summary、anomaly 或 diagnosis evidence。
- `BEFORE_NEXT_FLIGHT` 和 `IMMEDIATE_GROUNDING` maintenance recommendation 必须 `human_review_required=true`。
- audit 目录必须覆盖 flight-log-analysis、fault-diagnosis、maintenance-advisor 和 ops-report-generation。
- audit JSON 必须包含输入、输出、工具、规则、时间戳、状态和人工复核标记。
- `ops_report.md` 必须包含主要章节和证据线索。

## CLI 示例

```bash
drone-ops validate-report --report-dir data/sample_reports/
```

```bash
drone-ops validate-report \
  --summary data/sample_reports/flight_summary.json \
  --anomalies data/sample_reports/anomalies.json \
  --diagnosis data/sample_reports/diagnosis.json \
  --maintenance data/sample_reports/maintenance_recommendations.json \
  --report data/sample_reports/ops_report.md \
  --audit-dir data/sample_reports/audit
```

```bash
drone-ops validate-report --report-dir data/sample_reports/ --write-index
```

## 安全边界

本版本仍然是 offline advisory agent，不包含任何真实无人机控制能力：

- 不连接真实无人机。
- 不执行 MAVLink command。
- 不 arm/disarm。
- 不 takeoff/landing/RTL。
- 不 mission execution。
- 不 motor start。
- 不 firmware upload。
- 不 flight-controller parameter write。
- 不启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器。

`validate-report` 只读取本地报告产物和 audit JSON，用于离线证据链质量检查。任何维护或飞行安全建议仍必须由合格人员人工复核。

## 当前限制

- 当前 validation 以结构化 JSON 和轻量 Markdown 章节线索为主，不做脆弱的全文自然语言匹配。
- `report_validation.json` 为机器可读基线格式，后续仍可扩展 warning 分类、finding code 和更完整的 cross-reference graph。
- 该 PR 不包含 Web dashboard、fleet analytics、work order integration 或 report bundle。
