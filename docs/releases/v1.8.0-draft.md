# v1.8.0 - Offline Organization Handoff Package Baseline

Date: draft

Release status: draft

Release positioning: offline organization handoff package baseline on top of the v1.7.0 offline adapter and approval workflow release.

## 范围

v1.8.0 把 workspace、report bundle、dataset registry、offline adapter registry、approval packet 和 platform readiness checklist 收成一个本地组织级交接包 manifest。

本版本仍然是 offline-only、advisory-only，不连接真实无人机、不执行飞控命令、不接入真实维修系统或真实 fleet platform。

## Organization Handoff Package

新增能力：

- 新增 `OrganizationHandoffArtifact`。
- 新增 `OrganizationHandoffPackage`。
- 新增 sample fixture：`data/sample_handoff/organization_handoff_package.json`。
- artifact refs 只引用本地文件。
- artifact 和 package 默认 `human_review_required=true`。

## Handoff Validation CLI

新增能力：

- 新增 `packages.organization_handoff`。
- 新增 `validate-handoff-package` CLI。
- 输出 deterministic `handoff_validation.json`。
- 检查 required artifact 是否存在。
- 检查 reviewer roles、offline/advisory boundary、no external platform connection 和 no auto dispatch。

## 安全边界

v1.8.0 不包含：

- 真实无人机连接。
- MAVLink command execution。
- arm/disarm。
- takeoff / landing / RTL。
- mission execution。
- motor start。
- firmware upload。
- flight-controller parameter writing。
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接。
- 真实 fleet platform API 调用。
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用。
- 自动派单。
- 自动执行维修动作。
- 真实、敏感或未批准的二进制飞行日志。
