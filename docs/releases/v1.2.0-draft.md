# v1.2.0 - Local Web Dashboard MVP

Date: draft

Release status: draft

Release positioning: local read-only dashboard baseline on top of the v1.1.0
fleet health analytics release.

## 范围

v1.2.0 从 CLI 输出扩展到本地只读 Dashboard。它的定位是“离线运维复核入口”，不是实时飞控
平台，也不是远程多用户 SaaS。

本版本继续保持：

- offline-only
- mock/sample/sanitized local input only
- read-only dashboard behavior
- advisory-only
- human review required

## Dashboard Data Bundle Baseline

新增能力：

- 新增 `packages.dashboard` 本地 bundle builder。
- 新增 `dashboard-bundle` CLI。
- 输出确定性的 `dashboard_bundle.json`。
- Bundle 聚合 report、simulation、work order、fleet health、audit 和 evidence 本地 artifact reference。
- 新增 `docs/local_dashboard.md`。

这个阶段不启动 Web backend，也不提供 UI；它先为后续本地 backend 和 UI 提供稳定、可测试的数据源。

## 后续 v1.2.0 小阶段

## Local Read-only Backend Baseline

新增能力：

- 新增 `packages.dashboard.create_dashboard_app(bundle_path=...)`。
- 新增只读 API：`GET /health`。
- 新增只读 API：`GET /api/dashboard/bundle`。
- backend 只读取本地 `dashboard_bundle.json`，不接受写入请求。
- 新增 backend integration tests。

本阶段不远程部署、不引入多用户权限、不连接真实 fleet platform，也不启动或连接任何飞控/仿真器。

## 后续 v1.2.0 小阶段

## Minimal Web Dashboard UI and Readiness

新增能力：

- 新增 `GET /` 本地只读 Dashboard 页面。
- 页面读取 `/api/dashboard/bundle`。
- 页面展示 bundle id、可用板块、本地 artifact refs 和安全边界提示。
- 新增 `docs/v1.2.0_release_readiness.md`。
- 新增 v1.2.0 readiness docs test。

这个阶段仍然不引入远程部署、多用户权限、上传工作流或真实外部系统接入。

## 后续 v1.2.0 小阶段

1. 合并本地三个阶段为一个 GitHub PR。
2. PR 合并后发布 v1.2.0 tag / GitHub Release。

## 安全边界

v1.2.0 不包含：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- 真实 PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书或企业微信 API 调用
- 自动派单
- 自动执行维护动作
- 远程部署或多用户权限系统
- 真实、敏感或未经确认允许的二进制飞行日志
