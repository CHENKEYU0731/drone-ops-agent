# v1.9.0 - Offline Operations Platform Readiness Index

Date: draft

Release status: draft

Release positioning: offline platform readiness index baseline on top of the v1.8.0 organization handoff package release.

## 范围

v1.9.0 将 v0.7.0 到 v1.8.0 的本地离线能力收成一个 `platform_readiness_index.json`，用于展示当前项目已经具备哪些 offline-only / advisory-only 运维质量门禁能力，以及每项能力对应的推荐命令和输出文件。

本版本仍然不连接真实无人机、不执行飞控命令、不接入真实维修系统或真实 fleet platform。

## Platform Readiness Index

新增能力：

- 新增 `PlatformReadinessCapability`。
- 新增 `PlatformReadinessIndex`。
- 新增 sample fixture：`data/sample_platform/platform_readiness_index.json`。
- capability 按 `capability_id` 确定性排序。
- commands、output refs 和 safety notes 保持稳定排序。
- 所有 capability 和 index 默认 `human_review_required=true`。

## Platform Index Validation CLI

新增能力：

- 新增 `packages.platform_index`。
- 新增 `validate-platform-index` CLI。
- 输出 deterministic `platform_index_validation.json`。
- 写出 `audit/platform-readiness-index-validation-*.json`。
- 检查 required release checks、capability 命令、输出引用、人工复核字段和 offline/advisory 安全边界。

## 安全边界

v1.9.0 不包含：

- 真实无人机连接。
- MAVLink command execution。
- arm/disarm。
- takeoff / landing / RTL。
- mission execution。
- motor start。
- firmware upload。
- flight-controller parameter writing。
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接。
- 真实 fleet platform API 调用。
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用。
- 自动派单。
- 自动执行维护动作。
- 真实、敏感或未经批准的二进制飞行日志。
