# v1.6.0 - Offline Dataset and Case Management Baseline

Date: draft

Release status: draft

Release positioning: local dataset/case registry baseline on top of the v1.5.0 platform readiness release.

## 范围

v1.6.0 把仓库内已有 sample logs、report validation case、simulation case、eval case、fleet manifest 和 platform readiness fixtures 统一登记为本地 dataset registry。

该版本仍然是 offline-only、advisory-only 的本地质量门禁版本，不下载数据，不上传数据，不连接真实平台、真实维修系统或真实无人机。

## Dataset Registry Contract Baseline

新增能力：

- 新增 `DatasetCase`。
- 新增 `DatasetRegistry`。
- 新增 deterministic sample registry：`data/sample_datasets/registry.json`。
- registry 覆盖 flight log、report directory、simulation case、eval case、fleet manifest 和 platform readiness case。

## Dataset Validation CLI

新增能力：

- 新增 `packages.dataset_registry`。
- 新增 `validate-datasets` CLI。
- 输出 deterministic `dataset_validation.json`。
- 检查本地 source refs 是否存在。
- 检查 `sanitized_status` 是否属于允许值。
- 检查 `recommended_commands` 和 `expected_outputs` 是否存在。
- 检查 offline-only / advisory-only 安全边界。

## 安全边界

v1.6.0 不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未批准的二进制飞行日志

所有 dataset registry 输出都默认 `human_review_required=true`，只用于本地质量门禁和人工复核材料。
