# v0.5.0-draft - Log Formats and Offline Simulation Validation

状态：草稿，尚未发布。

## 概述

v0.5.0 草稿整合 PX4 ULog 离线解析、ArduPilot BIN 离线解析和 simulation-validation 离线仿真结果导入验证。该版本继续保持 offline-first 和 advisory-only 安全边界，不连接真实无人机，也不执行任何真实飞控动作。

## 新增能力

- 新增统一日志解析器接口和 registry。
- `analyze-log --format auto` 支持 `.csv`、`.json`、`.ulg`、`.bin` 扩展名识别。
- 新增 PX4 ULog 离线解析适配器和 mock `.ulg` fixture。
- 新增 ArduPilot BIN 离线解析适配器和 mock `.bin` fixture。
- 新增可选 extras：`.[px4]`、`.[ardupilot]`、`.[sim]`。
- 新增 `validate-simulation` CLI。
- 新增 offline simulation scenario/result 样例数据。
- 新增 `simulation_run.json` 输出和 `simulation-validation` audit JSON。
- 新增 ULog/BIN/SITL 相关单元测试和集成测试。

## 已有能力保持

- `run-mvp`
- `preflight-check`
- `monitor-replay`
- Markdown 运维报告
- PDF 报告导出
- evidence_refs
- audit JSON

## 安全边界

本版本仍然不包含任何真实无人机连接或控制能力：

- 不连接真实无人机。
- 不 arm/disarm。
- 不启动电机。
- 不起飞、降落、返航。
- 不执行航线。
- 不上传固件。
- 不写入飞控参数。
- 不执行 MAVLink command。

PX4 ULog、ArduPilot BIN 和 simulation validation 都只读取本地文件，输出风险、证据和审计记录，供人工复核。

## 验证计划

- `pytest`
- `run-mvp` CSV 回归
- `analyze-log --format auto` CSV
- `analyze-log --format auto` JSON
- `analyze-log --format auto` PX4 mock `.ulg`
- `analyze-log --format auto` ArduPilot mock `.bin`
- `validate-simulation` mock/offline flow
- `preflight-check` 回归
- `monitor-replay` 回归
- `export-pdf` 回归
- ULog/BIN 可选依赖缺失错误提示验证
- 安全边界关键词扫描

## 当前限制

- PX4 ULog 初版只覆盖最小可用 topic/字段映射。
- ArduPilot BIN 初版只覆盖最小可用 message/字段映射。
- SITL validation 初版只做 mock/offline simulation result import，不启动 PX4、ArduPilot 或 Gazebo。
- `.[sim]` 当前保留为空，用于后续仿真工具扩展。
- 报告对 simulation validation section 的深度集成仍可继续增强。
