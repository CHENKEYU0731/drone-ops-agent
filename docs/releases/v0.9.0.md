# v0.9.0 - Work Order Drafts

Date: 2026-06-26

Release status: ready

Release positioning: offline work order draft loop on top of the v0.8.0 report
integration release.

Release commit:

- `bf10158f20fd42bffff06823168b5db8f189c058`

## 范围

v0.9.0 把维护建议转成可审计、待人工确认的工单草稿，并补齐工单草稿质量门禁和报告展示路径。该版本继续保持 offline-only、advisory-only 定位，不接入真实工单系统，不自动派单，也不执行任何维护动作。

本地 release readiness checklist：

- `docs/v0.9.0_release_readiness.md`

## Work Order Drafts Baseline

新增能力：

- 新增 `WorkOrderDraft` schema。
- 新增 `packages.work_orders`，将 `MaintenanceRecommendation` 转换为工单草稿。
- 新增 CLI：`generate-work-orders`。
- 输出 `work_order_drafts.json`。
- 输出中文 Markdown 草稿 `work_order_drafts.md`。
- 输出 `audit/work-order-drafting-*.json`，记录输入、输出、规则和人工复核要求。

每条工单草稿包含：

- `work_order_id`
- `asset_id`
- `component`
- `priority`
- `action`
- `reason`
- `evidence_refs`
- `required_approval`
- `estimated_effort`
- `reviewer`
- `status`
- `source_recommendation_id`
- `human_review_required=true`

## Work Order Validation Baseline

新增能力：

- 新增 `validate-work-orders` CLI。
- 校验本地 `work_order_drafts.json`。
- 输出 `work_order_validation.json`。
- 输出 `audit/work-order-validation-*.json`。
- 检查每条工单草稿是否包含 `work_order_id`、`asset_id`、`source_recommendation_id`、`evidence_refs`、`required_approval`、`estimated_effort`、`status=DRAFT` 和 `human_review_required=true`。

该能力只做离线质量门禁，不接入真实维修系统，不自动派单，也不代表维护授权。

## Work Order Report Integration Baseline

新增能力：

- `generate-report` 新增可选参数 `--work-orders`。
- `generate-report` 新增可选参数 `--work-order-validation`。
- `ops_report.md` 可展示 `## 7.9 工单草稿`。
- `ops_report.md` 可展示 `## 7.10 工单验证`。
- 报告中明确工单草稿不会自动派单，不会连接真实维修系统，也不代表维护授权。

## CLI

```bash
drone-ops generate-work-orders \
  --maintenance data/sample_reports/maintenance_recommendations.json \
  --asset data/sample_assets/uav_001.json \
  --out data/sample_reports/
```

```bash
drone-ops validate-work-orders \
  --drafts data/sample_reports/work_order_drafts.json \
  --out data/sample_reports/
```

```bash
drone-ops generate-report \
  --summary data/sample_reports/flight_summary.json \
  --anomalies data/sample_reports/anomalies.json \
  --diagnosis data/sample_reports/diagnosis.json \
  --maintenance data/sample_reports/maintenance_recommendations.json \
  --work-orders data/sample_reports/work_order_drafts.json \
  --work-order-validation data/sample_reports/work_order_validation.json \
  --out data/sample_reports/ops_report.md
```

未安装 CLI entry point 时，可以使用 `python -m apps.cli.main ...` 执行同等命令。

## 安全边界

v0.9.0 继续保持离线建议系统定位。

本版本不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- 真实 PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 CMMS、Jira、飞书或企业微信 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或二进制飞行日志

所有工单输出均为 `DRAFT`，必须由合格人员人工复核和确认。验证通过只表示草稿结构和证据链满足离线质量门禁，不代表真实派单或维护授权。
