# v2.0.0 - Offline Operations Platform Baseline

Date: draft

Release status: draft

Release positioning: offline organization-level operations platform baseline on top of the v1.9.0 platform readiness index release.

## 范围

v2.0.0 将 v0.7.0 到 v1.9.0 的本地离线能力收成一个 `operations_platform_baseline.json`，用于展示当前项目已经具备组织级离线运维决策平台的基本结构、验证命令、预期输出、复核角色和安全边界。

本版本仍然不连接真实无人机、不执行飞控命令、不接入真实维修系统或真实 fleet platform。

## Operations Platform Baseline

新增能力：

- 新增 `OperationsPlatformModule`。
- 新增 `OperationsPlatformBaseline`。
- 新增 sample fixture：`data/sample_platform/operations_platform_baseline.json`。
- modules 按 `module_id` 确定性排序。
- artifact refs、validation commands、expected outputs、reviewer roles 和 safety notes 保持稳定排序。
- 所有 module 和 baseline 默认 `human_review_required=true`。

## Operations Platform Validation CLI

新增能力：

- 新增 `packages.operations_platform`。
- 新增 `validate-operations-platform` CLI。
- 输出 deterministic `operations_platform_validation.json`。
- 写出 `audit/operations-platform-validation-*.json`。
- 检查 release checks、module artifact refs、validation commands、expected outputs、reviewer roles、人工复核字段和 offline/advisory 安全边界。

## 安全边界

v2.0.0 不包含：

- 真实无人机连接。
- MAVLink command execution。
- arm/disarm。
- takeoff / landing / RTL。
- mission execution。
- motor start。
- firmware upload。
- flight-controller parameter writing。
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接。
- 真实 fleet platform API 调用。
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用。
- 自动派单。
- 自动执行维护动作。
- 真实、敏感或未经批准的二进制飞行日志。
