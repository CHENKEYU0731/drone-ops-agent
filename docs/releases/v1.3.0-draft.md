# v1.3.0 - Rule Pack and Skill Versioning

Date: draft

Release status: draft

Release positioning: offline rule governance and skill version metadata baseline on top of the v1.2.0 local dashboard release.

## 范围

v1.3.0 引入规则包、规则版本和 skill registry 的本地治理能力。它仍然只是 offline-only、
advisory-only 的审计与验证增强，不是飞控执行系统，也不是外部平台集成。

## Rule Pack Contract Baseline

新增能力：

- 新增 `RulePackScope`。
- 新增 `RulePackRule`。
- 新增 `RulePack`。
- 新增 deterministic sample rule pack：`data/sample_rule_packs/offline_default_rules.json`。
- 新增 `docs/rule_pack_versioning.md`。

## 后续 v1.3.0 小阶段

## Rule Pack Validation CLI

新增能力：

- 新增 `packages.rule_packs` 本地验证模块。
- 新增 `validate-rule-pack` CLI。
- 输出确定性的 `rule_pack_validation.json`。
- validation findings 暴露缺失 evidence fields、缺失 inputs 等治理问题。

## 后续 v1.3.0 小阶段

## Skill Version Registry and Readiness

新增能力：

- 新增本地 skill registry builder。
- 新增 `list-skills` CLI。
- 新增 `list-rule-packs` CLI。
- 输出确定性的 `skill_registry.json` 和 `rule_packs.json`。
- 新增 `docs/v1.3.0_release_readiness.md`。

这些输出只是离线治理元数据，不会连接真实无人机、外部平台或维修系统。

## 后续 v1.3.0 小阶段

1. 合并本地三个阶段为一个 GitHub PR。
2. PR 合并后发布 v1.3.0 tag / GitHub Release。

## 安全边界

v1.3.0 不包含：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- 真实 PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书或企业微信 API 调用
- 自动派单
- 自动执行维护动作
