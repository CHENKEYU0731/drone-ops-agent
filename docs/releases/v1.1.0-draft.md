# v1.1.0 - Fleet Health Analytics

Date: draft

Release status: draft

Release positioning: offline fleet health analytics baseline on top of the
v1.0.0 stable offline ops agent.

## 范围

v1.1.0 从单次飞行运维复盘扩展到多资产、多任务的离线健康趋势分析。

本版本仍然保持：

- offline-only
- mock/sample/sanitized local input only
- advisory-only
- human review required

## Fleet Analytics Contract Baseline

新增文档：

- `docs/fleet_health_analytics.md`

新增 schema：

- `FleetRiskLevel`
- `FleetAsset`
- `FleetHealthFinding`
- `FleetHealthSummary`

这些对象用于描述本地机队资产集合、机队级健康发现、风险等级、风险排序和证据追溯。

## 后续 v1.1.0 小阶段

## Offline Fleet Aggregation Baseline

新增能力：

- 新增 `packages.fleet_health` 本地聚合模块。
- 新增 `fleet-summary` CLI。
- 新增 `data/sample_fleet/fleet_manifest.json` 和 deterministic sample fleet fixtures。
- 输出 `fleet_health_summary.json`。
- 写出 `audit/fleet-health-analytics-*.json`。

当前聚合覆盖低电量、GPS 降级、振动异常和高优先级维护建议信号。所有结果仍是本地离线建议，
并保留 `human_review_required=true` 和 evidence refs。

## 后续 v1.1.0 小阶段

## Fleet Report and Readiness

新增能力：

- `fleet-summary --markdown <path>` 可写出本地 `fleet_health_report.md`。
- 新增 `docs/v1.1.0_release_readiness.md`。
- 新增 fleet report 和 release readiness focused tests。

`fleet_health_report.md` 展示机队执行摘要、风险排名、机队级发现、数据来源和安全边界。
它仍然只是本地离线复核材料，不代表真实飞行授权或真实维护授权。

## 后续 v1.1.0 小阶段

后续阶段计划：

## 安全边界

v1.1.0 不包含：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- 真实 PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书或企业微信 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未经确认允许的二进制飞行日志
