# v1.1.0 Release Readiness Checklist

这份清单用于收口 `v1.1.0 - Fleet Health Analytics`。它验证机队健康趋势分析仍然是本地、
确定性、offline-only 和 advisory-only 的离线分析能力。

## 范围

v1.1.0 release readiness 覆盖：

- fleet analytics schema contract。
- 本地 sample fleet manifest。
- offline fleet aggregation。
- `fleet_health_summary.json`。
- `fleet_health_report.md`。
- fleet health audit JSON。
- GitHub Actions Python 3.11 / Python 3.12。

v1.1.0 不连接真实无人机，不读取实时遥测，不连接真实 fleet platform，不自动派单。

## 必跑本地检查

运行全量测试：

```bash
pytest
```

运行 fleet focused tests：

```bash
pytest tests/unit/test_fleet_health_contracts.py \
  tests/unit/test_fleet_health_aggregation.py \
  tests/unit/test_fleet_health_report.py \
  tests/unit/test_v1_1_release_readiness_docs.py \
  tests/integration/test_fleet_health_cli.py
```

## Fleet Summary CLI 检查

生成本地机队健康摘要：

```bash
python -m apps.cli.main fleet-summary \
  --manifest data/sample_fleet/fleet_manifest.json \
  --out <tmp-fleet-dir> \
  --markdown <tmp-fleet-dir>/fleet_health_report.md
```

确认生成：

- `<tmp-fleet-dir>/fleet_health_summary.json`
- `<tmp-fleet-dir>/fleet_health_report.md`
- `<tmp-fleet-dir>/audit/fleet-health-analytics-*.json`

确认 `fleet_health_summary.json` 包含：

- `fleet_id`
- `asset_count`
- `flight_count`
- `highest_risk`
- deterministic `risk_rankings`
- `findings`
- `evidence_refs`
- `source_refs`
- `human_review_required=true`

确认 `fleet_health_report.md` 包含：

- `# 机队健康趋势报告`
- `## 2. 风险排名`
- `## 3. 机队级发现`
- `## 5. 安全边界`

## 安全边界

v1.1.0 仍然是 offline advisory release。

不得新增：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未经确认允许的二进制飞行日志
