# v1.2.0 Release Readiness

这份清单用于收口 `v1.2.0 - Local Web Dashboard MVP`。它验证 Dashboard 仍然是本地、
只读、离线复核入口，不代表真实飞行授权、真实维护授权或远程平台上线。

pytest

## Dashboard Bundle 检查

生成本地 dashboard bundle：

```bash
python -m apps.cli.main dashboard-bundle \
  --report-dir data/sample_reports \
  --fleet-summary data/sample_reports/fleet_health_summary.json \
  --fleet-report data/sample_reports/fleet_health_report.md \
  --out <tmp>/dashboard_bundle.json
```

确认：

- `dashboard_bundle.json` 已生成。
- `schema_version` 稳定。
- `bundle_id` 为 `DASHBOARD-BUNDLE-OFFLINE`。
- artifact refs 覆盖 report、simulation、work orders、fleet health、audit 和 evidence。
- `human_review_required=true`。
- safety boundary 标记包含 offline-only、read-only 和 advisory-only。

## Local Backend 检查

backend factory：

- `packages.dashboard.create_dashboard_app(bundle_path=...)`

只读 API：

- `GET /health`
- `GET /api/dashboard/bundle`

确认：

- `GET /health` 返回 `offline-read-only`。
- `GET /api/dashboard/bundle` 只读取本地 `dashboard_bundle.json`。
- `POST /api/dashboard/bundle` 不被允许。
- backend 不连接真实无人机、真实 fleet platform、真实维修系统或外部仿真器。

## 安全边界

v1.2.0 仍然不包含：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 远程部署
- 多用户权限系统
- 真实、敏感或未经确认允许的二进制飞行日志
