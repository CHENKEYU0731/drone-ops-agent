# v1.7.0 Release Readiness

这份清单用于收口 `v1.7.0 - Offline Adapter and Approval Workflow Baseline`。本版本把 v1.5.x 平台化准备里的 offline adapter registry 和 approval workflow validation 固化为本地质量门禁，但仍然只是 offline-only、advisory-only 的人工复核辅助能力。

## 范围

v1.7.0 覆盖：

- `OfflineAdapterRegistry`
- `ApprovalPacket`
- `data/sample_adapters/offline_adapter_registry.json`
- `data/sample_approvals/approval_packet.json`
- `validate-adapters`
- `validate-approvals`
- `adapter_validation.json`
- `approval_validation.json`
- validation audit records

## 本地验证命令

运行全量测试：

```bash
pytest
```

运行 focused tests：

```bash
pytest \
  tests/unit/test_offline_adapter_contracts.py \
  tests/unit/test_adapter_approval_validation.py \
  tests/integration/test_adapter_approval_cli.py \
  tests/unit/test_v1_7_release_readiness_docs.py
```

验证 adapter registry：

```bash
python -m apps.cli.main validate-adapters \
  --registry data/sample_adapters/offline_adapter_registry.json \
  --out <tmp>/adapter_validation.json
```

确认：

- `adapter_validation.json` 已生成。
- status 为 `PASS`。
- findings 为 `0`。
- safety boundary 显示 `offline_only=true` 和 `advisory_only=true`。
- 输出目录存在 `audit/offline-adapter-validation-*.json`。

验证 approval packet：

```bash
python -m apps.cli.main validate-approvals \
  --packet data/sample_approvals/approval_packet.json \
  --out <tmp>/approval_validation.json
```

确认：

- `approval_validation.json` 已生成。
- status 为 `PASS`。
- findings 为 `0`。
- required reviewer roles 已覆盖。
- 输出目录存在 `audit/approval-workflow-validation-*.json`。

## 安全边界

v1.7.0 不连接真实无人机，不执行 MAVLink command execution，不执行 arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing。

本版本也不启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器，不接入真实维修系统，不调用真实 fleet platform / CMMS / Jira / 飞书 / 企业微信 API，不自动派单，不提交真实、敏感或未批准的二进制飞行日志。

所有 adapter validation 和 approval validation 输出都默认 `human_review_required=true`，只能作为本地离线质量门禁和人工复核材料。
