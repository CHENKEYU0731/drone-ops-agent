# v0.7.0 Release Readiness Checklist

This checklist closes the v0.7.0 quality-gate work around report validation and
offline simulation validation. It is intentionally local, deterministic, and
advisory-only.

## Scope

v0.7.0 release readiness covers:

- Report validation and deterministic evidence indexing.
- Offline/mock simulation scenario matrix validation.
- Offline/mock simulation operational rule results with evidence references.
- Existing MVP, PDF, preflight, monitoring, PX4 mock, ArduPilot mock, and real
  fixture placeholder regression tests through `pytest`.
- GitHub Actions on Python 3.11 and Python 3.12.

It does not add real drone control, real simulator launch, real hardware
connection, work-order integration, fleet analytics, web dashboard, or v0.8.0
report bundle behavior.

## Required Local Checks

Run the full test suite:

```bash
pytest
```

Generate a temporary report directory:

```bash
python -m apps.cli.main run-mvp \
  --log data/sample_logs/example_flight.csv \
  --asset data/sample_assets/uav_001.json \
  --out <tmp-report-dir>
```

Validate the report evidence chain and write the deterministic index artifacts:

```bash
python -m apps.cli.main validate-report \
  --report-dir <tmp-report-dir> \
  --write-index
```

Confirm these files exist:

- `<tmp-report-dir>/evidence_index.json`
- `<tmp-report-dir>/report_validation.json`

Validate the offline/mock simulation fixture:

```bash
python -m apps.cli.main validate-simulation \
  --scenario data/sample_simulation/example_scenario.json \
  --result data/sample_simulation/example_simulation_result.json \
  --out <tmp-simulation-dir>
```

Confirm these files exist:

- `<tmp-simulation-dir>/simulation_run.json`
- `<tmp-simulation-dir>/audit/simulation-validation-*.json`

Confirm `simulation_run.json` includes:

- top-level `status`
- top-level `human_review_required=true`
- top-level `evidence_refs`
- deterministic `rule_results`
- rule-level `evidence_refs`
- rule-level `human_review_required=true`

Run the focused simulation matrix tests when reviewing simulation readiness:

```bash
pytest tests/unit/test_simulation_scenario_matrix.py \
  tests/unit/test_simulation_validation.py \
  tests/integration/test_simulation_validation_flow.py
```

## GitHub Actions Gate

Before publishing v0.7.0, confirm the latest `main` GitHub Actions run is
successful for:

- Python 3.11
- Python 3.12

The existing CI already runs `pytest` and smoke checks for core CLI flows. Avoid
expanding CI unless a future PR introduces a new release-critical behavior that
is not covered by the current suite.

## Safety Boundary

v0.7.0 remains an offline advisory quality-gate release.

Do not add:

- Real drone connection.
- MAVLink command execution.
- Arm/disarm.
- Takeoff, landing, or RTL.
- Mission execution.
- Motor start.
- Firmware upload.
- Flight-controller parameter writing.
- PX4, ArduPilot, Gazebo, SITL, or external simulator startup or connection.
- Real, sensitive, or binary flight logs.

Simulation validation remains mock-import-only. Report validation reads local
report artifacts and audit JSON only. Safety and maintenance recommendations
remain subject to human review.
