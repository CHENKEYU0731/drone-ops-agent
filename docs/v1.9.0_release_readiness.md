# v1.9.0 Release Readiness

这份清单用于收口 `v1.9.0 - Offline Operations Platform Readiness Index`。本版本把已有离线能力整理成一个可验证的平台 readiness index，但仍然只是 offline-only / advisory-only 的本地质量门禁。

## 范围

v1.9.0 覆盖：

- `PlatformReadinessCapability`
- `PlatformReadinessIndex`
- `data/sample_platform/platform_readiness_index.json`
- `validate-platform-index`
- `platform_index_validation.json`
- validation audit records
- `docs/platform_readiness_index.md`

## 本地验证命令

运行全量测试：

```bash
pytest
```

运行 focused tests：

```bash
pytest \
  tests/unit/test_platform_readiness_index_contracts.py \
  tests/unit/test_platform_readiness_index_validation.py \
  tests/integration/test_platform_index_cli.py \
  tests/unit/test_v1_9_release_readiness_docs.py
```

验证 platform readiness index：

```bash
python -m apps.cli.main validate-platform-index \
  --index data/sample_platform/platform_readiness_index.json \
  --out <tmp>/platform_index_validation.json
```

确认：

- `platform_index_validation.json` 已生成。
- status 为 `PASS`。
- findings 为 `0`。
- capabilities 数量稳定。
- 输出目录存在 `audit/platform-readiness-index-validation-*.json`。
- 所有 capability 都包含推荐命令、输出引用、安全说明和 `human_review_required=true`。

## 安全边界

v1.9.0 不连接真实无人机，不执行 MAVLink command execution，不执行 arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing。

本版本也不启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器，不接入真实维修系统，不调用真实 fleet platform / CMMS / Jira / 飞书 / 企业微信 API，不自动派单，不上传文件，不提交真实、敏感或未经批准的二进制飞行日志。

所有 platform index validation 输出都默认 `human_review_required=true`，只能作为本地离线质量门禁和人工复核材料。
