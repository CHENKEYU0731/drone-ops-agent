# v1.0.0 Release Readiness Checklist

这份清单用于收口 `v1.0.0 - Stable Offline Ops Agent`。它把 `v0.1.0` 到
`v0.9.0` 已完成的离线能力统一验收为第一个可靠主版本。

本清单是本地、确定性、offline-only、mock-import-only 和 advisory-only 的发布前检查。
它不代表真实飞行授权，也不代表真实维护授权。

## 范围

v1.0.0 release readiness 覆盖：

- 稳定 CLI contract
- 稳定 JSON 输出和报告输出
- 稳定 schema surface
- audit record 生成和消费契约
- report validation / evidence index
- offline/mock simulation validation 和 scenario matrix
- operational simulation rules 和 rule-level evidence
- work order drafts 和 work order validation
- PDF 报告导出
- safety regression gate
- GitHub Actions Python 3.11 / Python 3.12

v1.0.0 不新增大功能，不做 v1.1.0 fleet analytics，不做 local web dashboard，不接入真实
CMMS / Jira / 飞书 / 企业微信，也不启动或连接真实仿真器。

## 必跑本地检查

运行全量测试：

```bash
pytest
```

运行 v1.0 安全回归门禁：

```bash
pytest tests/unit/test_v1_safety_regression_gate.py
```

运行核心质量门禁 focused tests：

```bash
pytest tests/unit/test_report_validation.py \
  tests/integration/test_report_validation_cli.py \
  tests/unit/test_simulation_validation.py \
  tests/unit/test_simulation_scenario_matrix.py \
  tests/integration/test_simulation_validation_flow.py \
  tests/unit/test_work_order_drafts.py \
  tests/unit/test_work_order_validation.py \
  tests/integration/test_work_order_cli.py \
  tests/integration/test_work_order_validation_cli.py
```

## 临时报表目录检查

生成临时报表目录：

```bash
python -m apps.cli.main run-mvp \
  --log data/sample_logs/example_flight.csv \
  --asset data/sample_assets/uav_001.json \
  --out <tmp-report-dir>
```

确认生成：

- `<tmp-report-dir>/flight_summary.json`
- `<tmp-report-dir>/anomalies.json`
- `<tmp-report-dir>/diagnosis.json`
- `<tmp-report-dir>/maintenance_recommendations.json`
- `<tmp-report-dir>/ops_report.md`
- `<tmp-report-dir>/audit/*.json`

验证报告证据链并写出确定性索引：

```bash
python -m apps.cli.main validate-report \
  --report-dir <tmp-report-dir> \
  --write-index
```

确认生成：

- `<tmp-report-dir>/evidence_index.json`
- `<tmp-report-dir>/report_validation.json`

## Simulation Validation 检查

导入 offline/mock simulation fixture：

```bash
python -m apps.cli.main validate-simulation \
  --scenario data/sample_simulation/example_scenario.json \
  --result data/sample_simulation/example_simulation_result.json \
  --out <tmp-simulation-dir>
```

确认生成：

- `<tmp-simulation-dir>/simulation_run.json`
- `<tmp-simulation-dir>/audit/simulation-validation-*.json`

确认 `simulation_run.json` 包含：

- top-level `status`
- top-level `human_review_required=true`
- top-level `evidence_refs`
- deterministic `rule_results`
- rule-level `evidence_refs`
- rule-level `human_review_required=true`

## Work Order Draft 检查

基于临时报表目录生成本地工单草稿：

```bash
python -m apps.cli.main generate-work-orders \
  --maintenance <tmp-report-dir>/maintenance_recommendations.json \
  --asset data/sample_assets/uav_001.json \
  --out <tmp-report-dir>
```

确认生成：

- `<tmp-report-dir>/work_order_drafts.json`
- `<tmp-report-dir>/work_order_drafts.md`
- `<tmp-report-dir>/audit/work-order-drafting-*.json`

验证本地工单草稿：

```bash
python -m apps.cli.main validate-work-orders \
  --drafts <tmp-report-dir>/work_order_drafts.json \
  --out <tmp-report-dir>
```

确认生成：

- `<tmp-report-dir>/work_order_validation.json`
- `<tmp-report-dir>/audit/work-order-validation-*.json`

确认 `work_order_validation.json` 包含：

- `status=passed`
- `counts.validated_drafts`
- `counts.evidence_refs`
- `created_at=1970-01-01T00:00:00Z`

## 集成报告检查

将 simulation 和 work order 输出重新纳入 Markdown 报告：

```bash
python -m apps.cli.main generate-report \
  --summary <tmp-report-dir>/flight_summary.json \
  --anomalies <tmp-report-dir>/anomalies.json \
  --diagnosis <tmp-report-dir>/diagnosis.json \
  --maintenance <tmp-report-dir>/maintenance_recommendations.json \
  --simulation <tmp-simulation-dir>/simulation_run.json \
  --work-orders <tmp-report-dir>/work_order_drafts.json \
  --work-order-validation <tmp-report-dir>/work_order_validation.json \
  --out <tmp-report-dir>/ops_report.md
```

可选：导出 PDF：

```bash
python -m apps.cli.main export-pdf \
  --markdown <tmp-report-dir>/ops_report.md \
  --out <tmp-report-dir>/ops_report.pdf
```

确认报告仍是本地离线复核材料，不代表真实飞行授权或真实维护授权。

## GitHub Actions Gate

发布 v1.0.0 前，最终合并到 `main` 后需要确认 GitHub Actions 成功：

- Python 3.11: success
- Python 3.12: success

现有 CI 已运行 `pytest`。除非 v1.0.0 收口阶段新增 release-critical 行为，否则不要为了
v1.0.0 大幅改 CI workflow。

## 安全边界

v1.0.0 仍然是 offline advisory release。

不得新增：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未经确认允许的二进制飞行日志

所有报告、仿真验证、维护建议和工单草稿都必须保留 human review 语义。
