# v1.8.0 Release Readiness

这份清单用于收口 `v1.8.0 - Offline Organization Handoff Package Baseline`。本版本新增组织级离线交接包 manifest 和验证命令，但仍然只是 offline-only、advisory-only 的本地质量门禁。

## 范围

v1.8.0 覆盖：

- `OrganizationHandoffArtifact`
- `OrganizationHandoffPackage`
- `data/sample_handoff/organization_handoff_package.json`
- `validate-handoff-package`
- `handoff_validation.json`
- validation audit records

## 本地验证命令

运行全量测试：

```bash
pytest
```

运行 focused tests：

```bash
pytest \
  tests/unit/test_organization_handoff_contracts.py \
  tests/unit/test_organization_handoff_validation.py \
  tests/integration/test_organization_handoff_cli.py \
  tests/unit/test_v1_8_release_readiness_docs.py
```

验证 handoff package：

```bash
python -m apps.cli.main validate-handoff-package \
  --package data/sample_handoff/organization_handoff_package.json \
  --out <tmp>/handoff_validation.json
```

确认：

- `handoff_validation.json` 已生成。
- status 为 `PASS`。
- findings 为 `0`。
- artifacts 数量和 artifact types 稳定。
- 输出目录存在 `audit/organization-handoff-validation-*.json`。

## GitHub Actions

发布前确认 GitHub Actions Python 3.11 和 Python 3.12 均为 success。

## 安全边界

v1.8.0 不连接真实无人机，不执行 MAVLink command execution，不执行 arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing。

本版本也不启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器，不接入真实维修系统，不调用真实 fleet platform / CMMS / Jira / 飞书 / 企业微信 API，不自动派单，不提交真实、敏感或未批准的二进制飞行日志。

所有 handoff validation 输出都默认 `human_review_required=true`，只能作为本地离线质量门禁和人工复核材料。
