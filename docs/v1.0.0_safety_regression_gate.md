# v1.0.0 安全边界回归门禁

本文档定义 v1.0.0 的本地安全回归门禁。它用于确认项目从 `v0.1.0` 到 `v0.9.0`
累积的能力仍然保持 offline-only、mock-import-only 和 advisory-only，不会在 v1.0.0
收口阶段引入真实控制路径。

## 门禁目标

Safety Regression Gate 要证明：

- runtime code 没有真实无人机连接路径。
- runtime code 没有 MAVLink command execution。
- runtime code 没有 arm/disarm、takeoff、landing、RTL、mission execution 或 motor start。
- runtime code 没有 firmware upload 或 flight-controller parameter writing。
- runtime code 没有启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器。
- runtime code 没有调用真实 CMMS、Jira、飞书、企业微信或其他维修系统 API。
- work order drafts 仍然只是本地草稿，不自动派单，不执行维护动作。
- simulation validation 仍然只导入 offline/mock simulation result，不启动真实仿真器。

## 自动化测试

安全门禁由以下测试覆盖：

```bash
pytest tests/unit/test_v1_safety_regression_gate.py
```

该测试执行三类检查：

1. 扫描 `apps/` 和 `packages/` 下的 Python runtime code，禁止出现真实控制和网络调用入口。
2. 确认本安全门禁文档存在，并明确记录 offline-only / advisory-only 边界。
3. 确认 `docs/releases/v1.0.0-draft.md` 记录 Safety Regression Gate。

## 扫描范围

当前扫描范围聚焦运行时代码：

- `apps/**/*.py`
- `packages/**/*.py`

文档、测试和 fixture 中允许出现禁止动作的文字描述，因为这些内容用于说明边界、测试禁止项或记录历史。
如果未来新增新的 runtime root，应同步扩展 `tests/unit/test_v1_safety_regression_gate.py`。

## 禁止能力

v1.0.0 不包含以下能力：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未经确认允许的二进制飞行日志

## 人工复核要求

即使自动化安全扫描通过，v1.0.0 仍然是 advisory-only。任何飞行安全、仿真风险、
维护建议、工单草稿或报告结论都需要人工复核，不能作为真实飞行授权或自动维护授权。

## 发布前检查

发布 v1.0.0 前建议运行：

```bash
pytest
pytest tests/unit/test_v1_safety_regression_gate.py
```

如果安全门禁失败，应先确认是否真的新增了控制、网络或真实集成路径。除非经过单独设计、
review 和安全批准，否则不得通过扩大 allowlist 的方式绕过门禁。
