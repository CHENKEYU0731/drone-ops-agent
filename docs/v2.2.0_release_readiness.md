# v2.2.0 Release Readiness

这份清单用于收口 `v2.2.0 - Evaluation and Case Study Baseline`。本版本为现有离线仿真、诊断和报告质量门禁增加统一案例研究结果与中文报告。

## 本地验证

运行全量测试：

```bash
pytest
```

运行 focused tests：

```bash
pytest \
  tests/unit/test_case_study_runner.py \
  tests/integration/test_case_study_cli.py \
  tests/unit/test_simulation_scenario_matrix.py \
  tests/unit/test_eval_runner.py \
  tests/integration/test_demo_package_script.py \
  tests/unit/test_v2_2_release_readiness_docs.py
```

运行案例研究：

```bash
python -m apps.cli.main run-case-studies \
  --simulation-matrix data/sample_simulation/scenario_matrix.json \
  --eval-case data/sample_evals/diagnosis_report_eval_case.json \
  --out <tmp-case-study-dir>
```

确认：

- `case_study_results.json` 已生成。
- `case_study_report.md` 已生成并包含中文指标说明。
- `audit/evaluation-case-study-*.json` 已生成。
- 总案例数为 15，仿真场景数为 14。
- `PASS`、`REVIEW_REQUIRED`、`FAIL`、`INVALID_INPUT` 均被覆盖。
- `expected_status_accuracy=1.0`。
- `evidence_coverage_rate=1.0`。
- `false_alarm_count=0`。
- `missed_risk_count=0`。
- 重复运行得到相同 `result_digest`。
- 所有案例继续声明 `human_review_required=true`。

## 安全边界

本版本保持 offline-only、advisory-only 和 human-review-required，只读取 sample / mock / sanitized 本地文件并调用已有确定性规则。它不调用外部模型，不连接真实无人机、飞控、真实仿真器、维修系统或 fleet platform，不执行 MAVLink command、arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing。

所有结果仅用于离线案例研究和人工复核，不代表真实飞行安全、真实维修效果或统计泛化能力。
