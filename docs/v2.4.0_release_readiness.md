# v2.4.0 Release Readiness

这份清单用于收口 `v2.4.0 - Reproducible Distribution`。

## 自动测试

```bash
pytest
pytest tests/unit/test_distribution_tools.py tests/integration/test_distribution_scripts.py tests/unit/test_v2_4_release_readiness_docs.py
```

## 环境与约束

```bash
python scripts/check_environment.py --out <tmp>/environment.json
```

在受约束环境中使用 `--require-pass`，确认 `status=PASS`，Python 为 3.11+，核心依赖已安装且版本满足声明范围，输出不包含本地绝对路径或凭据。确认 `constraints/*.txt` 中所有直接依赖均使用精确版本。

## 构建与安装

Windows：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/verify_release.ps1 -OutDir dist/release-check
```

确认临时虚拟环境完成：受约束安装、全量 `pytest`、demo 成果生成、wheel/sdist 构建、wheel 重新安装和 `drone-ops --help` smoke test。

## 确定性发布包

连续运行两次：

```bash
python scripts/build_release_bundle.py --out <tmp>/first
python scripts/build_release_bundle.py --out <tmp>/second
```

确认两个 ZIP 的 SHA-256 一致；内部 `distribution_manifest.json` 和 `SHA256SUMS` 完整；只包含 Git 已跟踪文件，不包含 `.venv`、`dist`、cache、外部 ULog 二进制缓存、bytecode、符号链接、本地绝对路径或未跟踪文件。

## GitHub Actions

发布前确认 Python 3.11 和 Python 3.12 均为 success。现有 CI 的干净 runner 安装和全量测试继续作为跨版本门禁。

## 安全边界

构建/安装阶段可能显式访问 Python 包索引，但项目运行时代码不包含网络客户端。分析保持 offline-only、advisory-only 和 `human_review_required=true`；不连接真实无人机，不执行 MAVLink command，不启动或连接 PX4、ArduPilot、Gazebo、SITL，不写飞控参数，不上传固件，不接入真实维修或 fleet 系统。
