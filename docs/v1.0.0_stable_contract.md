# v1.0.0 稳定离线运维 Agent 契约

本文档冻结 `v1.0.0` 的本地 / 离线稳定契约。它覆盖项目从 `v0.1.0` 到
`v0.9.0` 已经形成的能力基线，用于把第一个离线无人机运维 Agent 主版本收成
可复查、可重复、可维护的稳定版本。

## 发布定位

`v1.0.0` 是稳定的 offline advisory operations agent baseline。它把现有
CLI、schema、报告、验证门禁、audit 记录和安全边界整理成可靠的本地工作流。

它不是飞控版本、不是仿真器控制版本，也不是真实维修系统集成版本。

## 稳定 CLI 命令

以下命令属于 v1.0 稳定离线契约。

| Command | Purpose | Stable inputs | Stable outputs |
| --- | --- | --- | --- |
| `run-mvp` | 运行核心 log -> diagnosis -> report 流程 | `--log`, `--asset`, `--out` | `flight_summary.json`, `anomalies.json`, `diagnosis.json`, `maintenance_recommendations.json`, `ops_report.md`, `audit/*.json` |
| `analyze-log` | 解析本地飞行日志并检测异常 | `--log`, `--asset`, `--out`, `--format` | `flight_summary.json`, `anomalies.json`, `audit/flight-log-analysis-*.json` |
| `diagnose` | 生成故障假设和维护建议 | `--summary`, `--asset`, `--out` | `diagnosis.json`, `maintenance_recommendations.json`, diagnosis 和 maintenance audit JSON |
| `generate-report` | 渲染 Markdown 运维报告 | `--summary`, `--anomalies`, `--diagnosis`, `--maintenance`, `--simulation`, `--work-orders`, `--work-order-validation`, `--out`, `--pdf`, `--asset` | `ops_report.md`, optional `ops_report.pdf`, `audit/ops-report-generation-*.json` |
| `export-pdf` | 将本地 Markdown 报告导出为 PDF | `--markdown`, `--out` | `ops_report.pdf` |
| `validate-report` | 验证报告证据链和 audit 覆盖 | `--report-dir` or explicit file paths, `--write-index` | optional `evidence_index.json`, `report_validation.json` |
| `preflight-check` | 运行离线飞行前检查 | `--asset`, `--battery`, `--mission`, `--observations`, `--rules`, `--out` | `preflight_check_result.json`, `audit/preflight-check-*.json` |
| `monitor-replay` | 用本地 telemetry 回放监控规则 | `--telemetry`, `--asset`, `--rules`, `--out` | `monitoring_summary.json`, `monitoring_events.json`, `audit/state-monitoring-*.json` |
| `validate-simulation` | 验证导入的 offline/mock simulation result | `--scenario`, `--result`, `--out` | `simulation_run.json`, `audit/simulation-validation-*.json` |
| `generate-work-orders` | 将维护建议转换为本地工单草稿 | `--maintenance`, `--asset`, `--out` | `work_order_drafts.json`, `work_order_drafts.md`, `audit/work-order-drafting-*.json` |
| `validate-work-orders` | 验证本地工单草稿质量 | `--drafts`, `--out` | `work_order_validation.json`, `audit/work-order-validation-*.json` |

稳定 JSON 输出需要保持足够确定性，便于 pytest 和 golden / snapshot 风格验证。

## 稳定 Audit 契约

主要 workflow 应写出或消费包含以下字段的 audit record：

- `skill_name`
- `skill_version`
- `input_refs`
- `output_refs`
- `tools_called`
- `rules_triggered`
- `human_review_required`
- `status`
- `created_at`

稳定 audit-producing skills 包括：

- `flight-log-analysis`
- `fault-diagnosis`
- `maintenance-advisor`
- `ops-report-generation`
- `preflight-check`
- `state-monitoring`
- `simulation-validation`
- `work-order-drafting`
- `work-order-validation`

Audit records 只保存在本地 JSON 文件中，不会发送到外部 logging、telemetry、工单或合规服务。

## 稳定安全边界

v1.0 继续保持 offline-only 和 advisory-only。

系统不得：

- 连接真实无人机
- 执行 MAVLink commands
- arm 或 disarm 飞行器
- 启动电机
- 起飞
- 降落
- 返航
- 执行任务或航点
- 上传固件
- 写入 flight-controller parameters
- 启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器
- 调用真实 CMMS、Jira、飞书、企业微信或其他维修系统 API
- 自动派发工单
- 自动执行维护动作
- 包含真实、敏感或未经确认允许的二进制飞行日志

Simulation validation 仍然是 mock-import-only。PX4 ULog 和 ArduPilot BIN 支持仍然是
offline parser paths，必须继续与 MAVLink command execution 严格分离。

## Experimental / Future Extension Areas

以下内容不属于 v1.0 稳定行为保证：

- 真实硬件连接
- 真实仿真器启动
- 真实 CMMS / work-order system integration
- 自动维护派单
- fleet analytics
- local web dashboard
- rule pack registry
- eval scoring framework
- organization-level reporting

这些能力可以在 v1.0 之后作为独立、受控的小版本加入，但必须保留上面的安全边界。

## v1.0 Readiness Implication

发布 v1.0 前，release readiness 应证明：

- 所有稳定 CLI 命令仍可运行，或被 pytest 覆盖
- 稳定 JSON 输出可以生成，并在适用位置被验证
- 报告可以包含稳定 optional sections，且没有回归
- 主要 workflow 会产出 audit records
- `tests/unit/test_v1_safety_regression_gate.py` 没有发现真实控制、真实仿真器或真实外部系统调用路径
- GitHub Actions 在支持的 Python 版本上通过
