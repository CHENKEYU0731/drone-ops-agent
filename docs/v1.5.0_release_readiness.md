# v1.5.0 Release Readiness

这份清单用于收口 `v1.5.0 - Platform Readiness`。本版本为后续 v2.0.0 平台化准备本地 workspace、report bundle、reviewer / approval model、offline adapter contract 和安全隐私治理边界，但仍然只是 offline-only、advisory-only 的离线能力。

## 本地验证命令

建议在发布前运行：

```bash
pytest
```

聚焦测试：

```bash
pytest \
  tests/unit/test_platform_readiness_contracts.py \
  tests/unit/test_platform_readiness.py \
  tests/integration/test_platform_readiness_cli.py \
  tests/unit/test_v1_5_release_readiness_docs.py
```

## 平台准备门禁

样例文件：

- `data/sample_platform/workspace_project.json`
- `data/sample_platform/report_bundle_manifest.json`
- `data/sample_platform/platform_readiness_checklist.json`

构建本地 report bundle manifest：

```bash
python -m apps.cli.main build-report-bundle \
  --report-dir <tmp-report-dir> \
  --workspace-project-id workspace-local-demo \
  --bundle-id bundle-local-demo \
  --drone-id UAV-001 \
  --out <tmp>/report_bundle_manifest.json
```

验证平台准备状态：

```bash
python -m apps.cli.main validate-platform-readiness \
  --workspace data/sample_platform/workspace_project.json \
  --bundle data/sample_platform/report_bundle_manifest.json \
  --checklist data/sample_platform/platform_readiness_checklist.json \
  --out <tmp>/platform_readiness_validation.json
```

确认：

- `report_bundle_manifest.json` 已生成。
- `platform_readiness_validation.json` 已生成。
- validation status 对 sample fixtures 为 `PASS`。
- 输出默认 `human_review_required=true`。
- 不发生外部上传、外部 API 调用、自动派单或真实飞控动作。

## 安全边界

v1.5.0 不连接真实无人机，不执行真实飞控命令，也不接入真实维修系统或真实 fleet platform。

本版本仍然不包含：

- 真实无人机连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
- 真实、敏感或未批准的二进制飞行日志

所有平台准备输出都只用于本地质量门禁和人工复核材料，不代表真实飞行授权、真实维修授权或真实平台接入授权。
