# v1.3.0 Release Readiness

这份清单用于收口 `v1.3.0 - Rule Pack and Skill Versioning`。它验证规则包、
规则版本和 skill registry 仍然只是本地离线治理能力，不代表真实飞控、真实仿真器或外部平台集成。

## Rule Pack 检查

样例规则包：

- `data/sample_rule_packs/offline_default_rules.json`

确认：

- `RulePackScope`、`RulePackRule`、`RulePack` schema 可加载。
- rule id 稳定且唯一。
- 每条 rule 包含 version、scope、inputs、thresholds、evidence fields 和 human review 标记。
- safety boundary 明确声明 offline-only 和 advisory-only。

## validate-rule-pack 检查

命令：

```bash
python -m apps.cli.main validate-rule-pack \
  --rule-pack data/sample_rule_packs/offline_default_rules.json \
  --out <tmp>/rule_pack_validation.json
```

确认：

- `rule_pack_validation.json` 已生成。
- `status` 对合法规则包为 `PASS`。
- validation findings 能暴露缺失 evidence fields 或缺失 inputs。
- 输出结构确定，适合 snapshot / golden 风格测试。

## Skill Registry 检查

命令：

```bash
python -m apps.cli.main list-skills \
  --rule-pack data/sample_rule_packs/offline_default_rules.json \
  --out <tmp>/skill_registry.json
```

```bash
python -m apps.cli.main list-rule-packs \
  --rule-pack data/sample_rule_packs/offline_default_rules.json \
  --out <tmp>/rule_packs.json
```

确认：

- `skill_registry.json` 已生成。
- core CLI skill 版本稳定。
- rule pack reference 包含 id、version 和 path。
- 输出为本地只读元数据，不触发任何规则执行或外部连接。

## 安全边界

本版本不连接真实无人机，不执行真实飞控命令，也不调用外部平台或维修系统。

v1.3.0 仍然不包含：

- 真实无人机连接
- 实时遥测连接
- MAVLink command execution
- arm/disarm
- takeoff / landing / RTL
- mission execution
- motor start
- firmware upload
- flight-controller parameter writing
- PX4、ArduPilot、Gazebo、SITL 或外部仿真器启动或连接
- 真实 fleet platform API 调用
- 真实 CMMS、Jira、飞书、企业微信或其他维修系统 API 调用
- 自动派单
- 自动执行维护动作
