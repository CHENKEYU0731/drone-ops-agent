# v2.5.1 最终对抗式审查

本次审查从恶意输入、误用、供应链和维护者四个视角检查 v2.5.0。所有确认的高/中风险阻塞项均已修复并加入回归测试。

## 已修复发现

### 高：managed 输出目录标记可伪造

旧逻辑只检查标记文件存在，demo 还接受 README 前缀，可能在用户指定错误输出路径时删除伪造目录。现在必须存在内容精确、非 symlink 的 managed marker；发布 ZIP 和 checksum 也拒绝 symlink 输出。

### 中：本地 Dashboard DOM 注入

旧页面使用 `innerHTML` 渲染 bundle 中的 artifact 名和值，恶意本地 JSON 可注入 HTML/event handler。现在只使用 DOM `textContent` 和节点追加。

### 中：超大输入与整文件 mock 探测

JSON、CSV、Markdown、ULog/BIN 缺少统一上限，PX4/ArduPilot parser 为识别 mock 曾读取整个二进制。现在增加 20 MiB JSON、10 MiB Markdown、250 MiB 飞行日志、250000 条记录和 10000 个 audit 文件上限；二进制只读取有界前缀，mock JSON 有界读取。

### 中：人工复核 fail-open

旧 report validator 只对高风险 diagnosis 和 grounding maintenance 强制人工复核，audit 也可声明 false。现在所有 diagnosis、maintenance 和 audit 均要求 `human_review_required=true`；开源 registry 不能自行设置 `real_world_flight_verified=true`。

### 中：质量门禁退出码不一致

部分 `validate-*` 命令和 `run-evals` 在 `REVIEW_REQUIRED` 时仍返回 0，自动化可能误判成功。现在所有这些质量门禁在非 `PASS` 时返回 1，同时保留结构化输出供人工复核。

### 低：CI 供应链与运行时弃用

CI 之前未使用 v2.4.0 直接依赖约束，并使用被 GitHub 标记为 Node 20 deprecated 的 action。现在安装使用 `constraints/dev.txt`，checkout/setup-python 固定到 v6 commit SHA。

## 残余风险

- `pyulog` 和 `pymavlink` 是第三方本地 parser，当前没有进程级沙箱；文件大小上限降低风险但不能替代依赖安全审计。
- 约束文件固定直接依赖，不是带全量传递依赖哈希的 lockfile。
- 常规 audit/source metadata 可能保留用户传入的本地路径以支持本地追溯；对外分享应使用会检查绝对路径的 portfolio showcase。
- 随机 run ID 和 audit 时间导致业务成果跨次生成的整体 ZIP 哈希变化；每个单次发布资产都有独立 manifest 和 SHA-256。
- 15 个 golden/mock 案例和 3 个上游 ULog 兼容结果不代表真实世界准确率、适航性或维修授权。
- 系统没有真实控制、外部写入或真实仿真器连接的威胁模型，因为这些能力明确不在项目范围内。

## 最终安全边界

系统保持 offline-only、advisory-only 和 human-review-required。不连接真实无人机，不执行 MAVLink command、arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing；不启动或连接 PX4、ArduPilot、Gazebo、SITL；不接入真实维修系统或 fleet platform。
