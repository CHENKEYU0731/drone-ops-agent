# v2.1.0 Release Readiness

这份清单用于收口 `v2.1.0 - Demo and Portfolio Readiness`。本版本把现有离线能力整理为可重复生成、便于检查和展示的本地成果包，同时强化危险输出路径与平台验证门禁。

## 范围

- `python scripts/generate_demo_outputs.py --out demo_outputs`
- 中文演示说明 `docs/demo_guide.md`
- 成果包中的 `ops_report.pdf`、`evidence_index.json`、`report_validation.json` 和 `simulation_run.json`
- 工单草稿、机队健康报告、本地只读 Dashboard 数据包和平台验证结果
- 输出目录管理标记 `.drone-ops-demo-output`
- `validate-platform-index` 与 `validate-operations-platform` 的稳定质量门禁
- 所有结论默认 `human_review_required=true`

- `demo_outputs/.drone-ops-demo-output` 存在。
- 未受管理的非空目录以及仓库根目录、用户目录等危险路径会被拒绝。

- 缺少必要模块或产物引用时，结果为 `REVIEW_REQUIRED`，CLI 返回非零退出码。

## 安全边界

本版本保持 offline-only 和 advisory-only，仅使用仓库内 sample / mock / sanitized fixture。它不连接真实无人机，不执行 MAVLink command，不执行 arm/disarm、takeoff、landing、RTL、mission execution、motor start、firmware upload 或 flight-controller parameter writing。

本版本不启动或连接 PX4、ArduPilot、Gazebo、SITL 或外部仿真器，不接入真实维修系统或 fleet platform，不自动派单，不上传文件，也不提交真实、敏感或未经批准的二进制飞行日志。所有输出只用于本地演示、离线质量检查和人工复核，不代表真实飞行或维护授权。
